
#### XSS攻擊主要可分為三類：
反射型（Reflected）、儲存型（Stored）和基於DOM的（DOM-based）。
```
反射型XSS：
是一種常見的網頁安全威脅。在這種攻擊中，攻擊者會試圖在URL中嵌入惡意的JavaScript腳本。
當目標用戶點擊此URL時，該腳本將在用戶的瀏覽器中執行。
舉例來說：
用戶可能會收到一封似乎合法的電子郵件，該郵件中包含一個似乎是合法的鏈接，但實際上已經被篡改。
當用戶點擊此鏈接時，嵌入在URL中的惡意腳本會在其瀏覽器中執行。
這可能導致攻擊者能夠竊取用戶的身份驗證令牌、進行非授權的操作，或甚至插入惡意內容。
```

```
儲存型XSS：
攻擊者將惡意腳本儲存在網站的資料庫中，例如在留言板、評論區等地方留下惡意腳本。
當其他用戶訪問相關頁面時，惡意腳本會自動執行。此類攻擊具有較高的危害性，因為它不需要用戶主動操作。
```

```
基於DOM的XSS：
攻擊者通過操作網站的Document Object Model（DOM）來執行惡意腳本。此類攻擊涉及網頁的客戶端腳本，
如JavaScript，經常利用瀏覽器的缺陷來實現攻擊

```

```
輸入驗證：對所有用戶輸入的數據進行嚴格的
驗證，包括表單、查詢字符串、Cookie等。限制輸入的類型、長度和範圍，避免接受不必要的數據。
此外，應使用強類型和正則表達式來驗證用戶輸入，確保數據符合預期格式。
```
```
輸出編碼：
在將用戶輸入的數據顯示到網頁上之前，應對特殊字符（如尖括號、單引號、雙引號等）進行轉譯，
避免瀏覽器將其解析為HTML或JavaScript代碼。

```
使用安全的HTTP頭部：
設置瀏覽器的安全HTTP頭部，如Content-Security-Policy（CSP）和X-XSS-Protection，
可以限制腳本的執行範圍並防止XSS攻擊。例如，CSP可以指定允許腳本從哪些來源加載，以防止外部惡意腳本的注入。
