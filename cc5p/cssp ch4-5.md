ch.4 雲數據安全

```
雲數據生命週期
--創建create
-->遠程創數據 上傳前加密避免中間人、雲數據內部威脅
->>被FIPS 140-2認可加密解決、實施密鑰管理辦法、最好使用IPSec VPN連接網路
-->雲創建數據 透過遠程操作創建數據時進行加密避免數據中心人不必要的訪問或查看、密鑰管理是最佳實踐
>>>>創建時應分類、分級、分配標間、標誌、記號和指定元數據

--儲存store 短期儲存
-->創建階段的活動同步、通過加密減緩在數據中心暴露的威脅

--使用use 操作和修改數據
-->連接都必須被安全保護、使用加密隧道、用戶使用BYOD利用培訓和使用安全使用技術DRM,DLP
-->所有者應注意限制修改和處理數據的權限、僅限執行分配任務所需功能
-->日記和審計是重要控制手段、雲提供者要確認VM數據不被同設備其他主機讀取和檢測
-->數據中心人不能訪問任何原始客戶數據

--共享 share 傳輸給其他用戶 根據司法管轄限制被發送到哪地，進出口限制
-->國際武器運輸條例ITAR 美國禁制國防方面出口，可包括加密系統
-->出口管理條例EAR  美國商務部禁止軍民兩用物品出口，技術可以適用於商業或軍事
-->密碼學cryptography  各國限制進口密碼系統和加密素材
-->某形式的數據輸出監測

--歸檔archive 長期
-->密碼學 cryptography 避免密鑰管理不善致額外暴露或全部損失
-->物理位置 location 環境因素、司法管轄權因素、距離歸檔位置有多遠、應急行動可否訪問數據、發勝事件可獲取數據
-->數據格式 format 介質攜帶防到、環境因素影響、需要保存多長、需要時能被正式環境硬件訪問使用？
-->工作人員 staff 人員控制措施、背調、可信、持續監測
-->流程procedure 需要時恢復數據、定期歸檔、多久一次全備份、差異備份頻率

--銷毀destroy  加密擦除crypto shredding是唯一可行和徹底方法

雲儲存架構
```
捲儲存：基於文件儲存和快儲存
--文件儲存 file 數據儲存和顯示與傳統文件結構一樣，隨個雲技術和大數據分析越來越流行
--塊儲存 block  文件儲存有文件夾和文件的層次結構、有個空白卷讓庫個放任何內容
>>>塊儲存更靈活性能高但是需要大量管理需要操作系統來儲存排序檢索數據，儲存包含多種形式比如企業備份服務
-->糾偏編碼 erasure coding冗余技術：實現在RAID數據保護解決方案，
>>>卷服務與IaaS服務結合

基於對象的儲存
>>>雲服務模型都可包含堆像儲存架構、但對象儲存通常與IaaS服務結合
-->儲存體架構允許進行大量描述野影加索引和數據策略執行能DRM和DLP的數據管理功能集中化
數據庫
-->根據本身特性和元素儲存在數據中心後端透過在驗應用和API訪問數據、常見配置與PaaS,SaaS一起工作
內容分發網路 CDN
-->
