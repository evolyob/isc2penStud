ch.4 雲數據安全

```
雲數據生命週期
--創建create
-->遠程創數據 上傳前加密避免中間人、雲數據內部威脅
->>被FIPS 140-2認可加密解決、實施密鑰管理辦法、最好使用IPSec VPN連接網路
-->雲創建數據 透過遠程操作創建數據時進行加密避免數據中心人不必要的訪問或查看、密鑰管理是最佳實踐
>>>>創建時應分類、分級、分配標間、標誌、記號和指定元數據

--儲存store 短期儲存
-->創建階段的活動同步、通過加密減緩在數據中心暴露的威脅

--使用use 操作和修改數據
-->連接都必須被安全保護、使用加密隧道、用戶使用BYOD利用培訓和使用安全使用技術DRM,DLP
-->所有者應注意限制修改和處理數據的權限、僅限執行分配任務所需功能
-->日記和審計是重要控制手段、雲提供者要確認VM數據不被同設備其他主機讀取和檢測
-->數據中心人不能訪問任何原始客戶數據
***
--共享 share 傳輸給其他用戶 根據司法管轄限制被發送到哪地，進出口限制
-->國際武器運輸條例ITAR 美國禁制國防方面出口，可包括加密系統
-->出口管理條例EAR  美國商務部禁止軍民兩用物品出口，技術可以適用於商業或軍事
-->密碼學cryptography  各國限制進口密碼系統和加密素材
-->某形式的數據輸出監測

--歸檔archive 長期
-->密碼學 cryptography 避免密鑰管理不善致額外暴露或全部損失
-->物理位置 location 環境因素、司法管轄權因素、距離歸檔位置有多遠、應急行動可否訪問數據、發勝事件可獲取數據
-->**數據格式 format 介質攜帶防到、環境因素影響、需要保存多長、需要時能被正式環境硬件訪問使用？
-->工作人員 staff 人員控制措施、背調、可信、持續監測
-->流程procedure 需要時恢復數據、定期歸檔、多久一次全備份、差異備份頻率

--銷毀destroy  加密擦除crypto shredding是唯一可行和徹底方法

```

雲儲存架構
```
捲儲存：基於文件儲存和快儲存
--文件儲存 file 數據儲存和顯示與傳統文件結構一樣，隨個雲技術和大數據分析越來越流行
--塊儲存 block  文件儲存有文件夾和文件的層次結構、有個空白卷讓庫個放任何內容
>>>塊儲存更靈活性能高但是需要大量管理需要操作系統來儲存排序檢索數據，儲存包含多種形式比如企業備份服務
-->糾偏編碼 erasure coding冗余技術：實現在RAID數據保護解決方案，
>>>卷服務與IaaS服務結合

基於對象的儲存
>>>雲服務模型都可包含堆像儲存架構、但對象儲存通常與IaaS服務結合
-->儲存體架構允許進行大量描述野影加索引和數據策略執行能DRM和DLP的數據管理功能集中化
數據庫
-->根據本身特性和元素儲存在數據中心後端透過在驗應用和API訪問數據、常見配置與PaaS,SaaS一起工作
內容分發網路 CDN
-->靠近使用頻率較高的地理位址，提供經常提供的數據副本，提高帶寬和交付質量

```
雲數據安全基本策略

```
加密技術
--> data in rest(儲存的),transit(傳輸的),use(使用中)
--> 保護自己的數據不會意外訪問彼此的數據
--密鑰管理
-->保護水平 level protection
->>加密密鑰保護要跟被加密數據保持相同或更高的安全水準，只要私鑰未被修漏才有效的。
-->密鑰恢復 key recovery
->>因丟失或其他特殊原因須恢復密鑰。此技術要獲得網問數據流程要涉及多人，每人只能有訪問密鑰的一部分
-->密鑰分發 key distribution
->>通常利用外通道傳遞但昂貴，密鑰永遠不可用明文形式傳遞
-->密鑰撤銷 key revocation
->>禁止用戶對企業訪問
-->密鑰託管 key escrow
->>受信任第三方持有密鑰副本，有助於其他許多密鑰管理工作
-->外包密鑰管理 outsourcing key mgmt
->>給特定員工保管或找雲訪問安全代理CASB提供IAM跟密鑰管理服務，但不要存在供應商那邊。

同態加密 homomorphic  允許不需要先解密狀況處理加密數據


遮蔽、混淆、匿名和標記技術
-->測試環境 sandbox 不產生正式數據，確認真實功能和性能的同特徵數據
-->最小權限  僅擁有任務所需許可和訪問，不暴露整體性。
-->安全遠程訪問 只顯示數據摘要不是全貌

數據模糊化
-->隨機化randomization  隨機替換部分或全部數據，僅保留其特性長度大小寫。
-->散裂hashing  數據模糊化不可恢復，也可對其進行完整性檢查，變成固定長度會丟失原始數據特性
-->打亂shuffling 同數據不同條目表示數據，但缺點是實際數據
-->遮蔽masking  用無用字符隱藏數據
-->空值null    顯示空值

=>>靜態: 創建一個新的的數據及作為原始數據副本，只有模糊化副本數據被使用
=>>動態: 數據需要使用時被模糊化
匿名
>>>標記技術 tokenization 一個真實敏感數據另一個抽象標記，需要調用時需要標記服務器驗證身分
```

```
SIEM

-->日誌數據集中採集 諸多來源中提取聚集無外處理沒有其他功能
--會成為攻擊非常有吸引力的目標、需要額外的安全控制層
-->增強分析能力  習以為常產生鬆懈、部分功能自動化但無法識別特定攻擊模型
--低頻緩慢行持續威脅(APT高持續性深透攻擊)、沒有顯著指標造成自動化工具無法發現
>>>儀表板 dashboarding 讓管理者簡明直觀掌握環境狀況
>>>自動響應   根據策略和環境進行編程、提供信息才能真正為提供價值

出口持續監測DLP  識別數據、監測活動、執行策略

-->額外安全性DLP 縱深防禦另外一個控制將計數據意外洩漏或惡意披露可能性最後一個機制
-->策略執行  當用戶執行違反策略行為、用戶將被DLP警告
-->加強持續監測  可設置DLP工具為持續監控套件提供更多日誌流、網路出口處或各端點
-->法律法規合規  DLP識別特定類型種類數據、控制數據傳播更好遵守強制監管規定
>>>DLP可連接到DRM工具而對允許對知識產權進行額外控制
>>>DLP 雲數據生命週期的創建階段對所有者指定數據進行分類分級及分配標籤和元數據
>>>但雲環境可能不允許堆數據中心環境有足夠訪問權線、使得配置和使用難以成功、成本高
>>>
```
